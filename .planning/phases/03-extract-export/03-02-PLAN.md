Refactor `CmdExport` in `export.go` to implement the full export pipeline producing a ZIP archive:

**Update ExportArgs:**
- Change `Output` default from `"knowledge.tar.gz"` to `"graph.zip"`
- Add `Input` field (alias for `From`, to support `--input` flag per success criteria)
- Add `SkipDiscovery` bool flag
- Add `LLMDiscovery` bool flag (opt-in, default off per research recommendation)
- Add `MinConfidence` float64 flag (default 0.5)
- Keep `Version`, `GitVersion`, `Publish` flags

**New ExportMetadata struct** (replaces KnowledgeMetadata for ZIP format):
```go
type ExportMetadata struct {
    Version           string    `json:"version"`
    SchemaVersion     int       `json:"schema_version"`
    CreatedAt         string    `json:"created_at"`       // ISO 8601
    ComponentCount    int       `json:"component_count"`
    RelationshipCount int       `json:"relationship_count"`
    InputPath         string    `json:"input_path"`
    IgnorePatterns    []string  `json:"ignore_patterns,omitempty"`
    AliasesApplied    int       `json:"aliases_applied"`
    Checksum          string    `json:"checksum,omitempty"`
}
```
Keep `KnowledgeMetadata` for backward compatibility (existing tar.gz import uses it).

**Refactored CmdExport pipeline:**

1. Parse args (updated ParseExportArgs with new flags)
2. Load `.graphmdignore` via `LoadGraphmdIgnore(absFrom)` → feed patterns into `ScanConfig`
3. Load alias config via `LoadAliasConfig(absFrom)`
4. Scan directory: `ScanDirectory(absFrom, config)` → `[]Document`
5. Build initial graph: create `NewGraph()`, add nodes from docs (same as cmdIndex pattern), run `NewExtractor(absFrom).Extract()` for link-based edges
6. Run component detection: `NewComponentDetector().DetectComponents(graph, docs)` → set `node.ComponentType`, build mentions
7. Apply aliases: iterate graph nodes and edges, resolve names via `AliasConfig.ResolveAlias()`. If a node ID changes due to aliasing, update edges accordingly. Count aliases applied.
8. Run discovery algorithms (if not skipped): call `DiscoverRelationships(docs, nil)` for discovered edges, filter by minConfidence, add to graph
9. Save to temporary SQLite: create temp DB path, `OpenDB(tmpDBPath)`, `SaveGraph(graph)`, `SaveComponentMentions(mentions)`, close DB
10. Build `ExportMetadata` with all required + extended fields. Use `time.Now().UTC().Format(time.RFC3339)` for created_at.
11. Package as ZIP using `archive/zip`:
    - Create output file (ensure `.zip` suffix)
    - Add `graph.db`: read temp DB file, write to zip as `"graph.db"`
    - Add `metadata.json`: marshal ExportMetadata with `json.MarshalIndent`, write to zip as `"metadata.json"`
12. Clean up temp DB
13. Print summary to stderr (archive path, size, component count, relationship count, elapsed time)
14. Publish to S3 if requested (update S3 path logic for .zip extension)

**Important:** Do NOT call `buildIndex()` — build the full pipeline directly (avoids double discovery). Do NOT include markdown files in the ZIP (user decision: only graph.db + metadata.json).

Update imports: add `"archive/zip"`, remove `"archive/tar"` and `"compress/gzip"` if no longer used elsewhere. Keep the tar helper functions (`addFileToTar`, `addBytesToTar`) since `importtar.go` may still reference them — only remove if confirmed unused.
