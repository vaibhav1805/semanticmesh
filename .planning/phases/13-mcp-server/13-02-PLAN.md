2. **Create `internal/mcp/server.go`:**

   ```go

   ```

   - `NewServer() *mcpsdk.Server` — Creates MCP server with Implementation{Name: "graphmd", Version: "2.0.0"}, registers all 5 tools via `registerTools(server)`, returns server.

   - `Run() error` — The main entry point called by CLI:
     a. `log.SetOutput(os.Stderr)` — redirect Go's default logger to stderr
     b. Save `origStdout := os.Stdout`
     c. `os.Stdout = os.Stderr` — redirect all stdout to stderr (stdout guard)
     d. Create server via `NewServer()`
     e. Restore `os.Stdout = origStdout` — SDK's StdioTransport needs real stdout
     f. Create signal context: `ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGTERM, syscall.SIGINT)`
     g. `defer stop()`
     h. `fmt.Fprintf(os.Stderr, "graphmd MCP server starting on stdio...\n")`
     i. Return `server.Run(ctx, mcpsdk.NewStdioTransport())` (check if it's `mcp.NewStdioTransport()` or `&mcp.StdioTransport{}` — SDK docs say `mcp.NewStdioTransport()` for recent versions)

   **Important:** The stdout guard/restore sequence ensures that any `fmt.Println` in tool registration or library init goes to stderr, while the transport gets the real stdout for JSON-RPC. Test this empirically — if the SDK captures stdout at NewServer time, adjust the sequence.

   Use import alias to avoid package name collision: `import mcpsdk "github.com/modelcontextprotocol/go-sdk/mcp"` since the local package is also named `mcp`.

3. **Create `internal/mcp/tools.go`:**

   Define 5 typed input structs with `json` and `jsonschema` struct tags for auto JSON Schema generation:

   ```go
   type ImpactArgs struct {
       Component     string  `json:"component" jsonschema:"required,description=The component to analyze for downstream impact"`
       Depth         int     `json:"depth,omitempty" jsonschema:"description=Traversal depth (default 1; use 0 for unlimited)"`
       MinConfidence float64 `json:"min_confidence,omitempty" jsonschema:"description=Minimum confidence threshold (0.0-1.0)"`
       SourceType    string  `json:"source_type,omitempty" jsonschema:"description=Filter by detection source: markdown or code or both"`
       Graph         string  `json:"graph,omitempty" jsonschema:"description=Named graph to query (default: most recent import)"`
   }

   type DependenciesArgs struct { /* same fields as ImpactArgs */ }

   type PathArgs struct {
       From          string  `json:"from" jsonschema:"required,description=Source component"`
       To            string  `json:"to" jsonschema:"required,description=Target component"`
       Limit         int     `json:"limit,omitempty" jsonschema:"description=Maximum paths to return (default 10)"`
       MinConfidence float64 `json:"min_confidence,omitempty" jsonschema:"description=Minimum confidence per hop (0.0-1.0)"`
       SourceType    string  `json:"source_type,omitempty" jsonschema:"description=Filter by detection source: markdown or code or both"`
       Graph         string  `json:"graph,omitempty" jsonschema:"description=Named graph to query (default: most recent import)"`
   }

   type ListArgs struct {
       Type          string  `json:"type,omitempty" jsonschema:"description=Filter by component type (service, database, cache, etc.)"`
       MinConfidence float64 `json:"min_confidence,omitempty" jsonschema:"description=Minimum confidence for connected edges (0.0-1.0)"`
       SourceType    string  `json:"source_type,omitempty" jsonschema:"description=Filter by detection source: markdown or code or both"`
       Graph         string  `json:"graph,omitempty" jsonschema:"description=Named graph to query (default: most recent import)"`
   }

   type GraphInfoArgs struct {
       Graph string `json:"graph,omitempty" jsonschema:"description=Named graph to query (default: most recent import)"`
   }
   ```

   - `registerTools(server *mcpsdk.Server)` — Registers all 5 tools using `mcpsdk.AddTool(server, toolDef, handler)`.

   Tool descriptions (invest in quality — these drive agent behavior):
   - `query_impact`: "Analyze downstream impact of a component failure. Returns all components that directly or transitively depend on the specified component, with confidence scores. Use this to answer 'if X fails, what breaks?'"
   - `query_dependencies`: "Find what a component depends on. Returns all upstream dependencies with confidence scores. Use this to answer 'what does X need to work?'"
   - `query_path`: "Find dependency paths between two components. Returns paths with per-hop confidence scores. Use this to answer 'how does X connect to Y?'"
   - `list_components`: "List all components in the dependency graph with optional type and confidence filters. Use this to explore the graph before targeted queries."
   - `graphmd_graph_info`: "Get metadata about the loaded dependency graph: name, version, component count, relationship count. Use this first to verify a graph is loaded and assess its scope."

   Each handler function:
   a. Converts MCP args to `knowledge.Query*Params`
   b. Calls `knowledge.Execute*Query(params)` or `knowledge.GetGraphInfo(params)`
   c. On error: type-assert to `*knowledge.QueryError`. If it is a QueryError (user error like NOT_FOUND, MISSING_ARG), return `CallToolResult{IsError: true}` with JSON-encoded error. If it's an infrastructure error, return it as the handler error (3rd return value).
   d. On success: JSON marshal the envelope/result, return as `CallToolResult` with `TextContent`.

   Error handling pattern (critical — see research pitfall #4):
   ```go
   envelope, err := knowledge.ExecuteImpactQuery(params)
   if err != nil {
       var qe *knowledge.QueryError
       if errors.As(err, &qe) {
           // User error — return as tool result
           errJSON, _ := json.Marshal(qe)
           return &mcpsdk.CallToolResult{
               Content: []mcpsdk.Content{mcpsdk.NewTextContent(string(errJSON))},
               IsError: true,
           }, nil
       }
       // Infrastructure error — return as handler error
       return nil, err
   }
   ```
