  <action>
Create two files in `internal/code/`:

**integration.go** — High-level entry point for CLI commands:
- `RunCodeAnalysis(dir string) ([]CodeSignal, error)`:
  1. Call `InferSourceComponent(dir)` to get source component name
  2. Create `NewCodeAnalyzer(sourceComponent)`
  3. Register `goparser.NewGoParser()` (and future parsers here)
  4. Call `analyzer.AnalyzeDir(dir)`
  5. Return signals
- `PrintCodeSignalsSummary(w io.Writer, signals []CodeSignal)` — prints a summary to stderr:
  - Count by detection_kind (e.g., "  3 HTTP calls, 2 DB connections, 1 cache client")
  - Total signal count
  - Source component name
  - Keep it concise — this is diagnostic output, not the graph

**integration_test.go** — End-to-end test:
- Create a temp directory with:
  - A `go.mod` file (`module github.com/example/myservice`)
  - A `main.go` file containing: `http.Get("http://payment-api/pay")`, `sql.Open("postgres", "postgres://mydb:5432/app")`, and a comment `// Depends on cache-cluster`
  - A `handler_test.go` file with some sql.Open calls (should be skipped)
- Call `RunCodeAnalysis(tmpDir)`
- Assert: 3 signals returned (http_call, db_connection, comment_hint)
- Assert: source component is "myservice" (from go.mod)
- Assert: handler_test.go signals are NOT included
- Assert: target_component values are correct ("payment-api", "mydb" or "postgres", "cache-cluster")

**export.go:**
1. Add `AnalyzeCode bool` field to `ExportArgs`
2. Add `fs.BoolVar(&a.AnalyzeCode, "analyze-code", false, "Analyze source code for infrastructure dependencies")` in ParseExportArgs
3. In `CmdExport`, after the markdown scan/discovery steps and before ZIP packaging:
   - If `a.AnalyzeCode` is true, call `code.RunCodeAnalysis(absFrom)`
   - Print summary to stderr via `code.PrintCodeSignalsSummary(os.Stderr, signals)`
   - Print `"  Code analysis: %d signals detected\n"` to stderr
   - (Signals are not yet stored in the graph — that's Phase 12)
4. Add import for `"github.com/graphmd/graphmd/internal/code"`

**crawl_cmd.go:**
1. Add `AnalyzeCode bool` to `CrawlArgs` struct
2. Add `--analyze-code` flag to ParseCrawlArgs
3. In `CmdCrawl`, if AnalyzeCode is set and the input is a directory (not a ZIP):
   - Call `code.RunCodeAnalysis(inputDir)` and print summary to stderr
   - (When input is a ZIP file, code analysis doesn't apply — can't analyze source from a graph archive)

**cmd/graphmd/main.go (cmdIndex):**

   - Call `code.RunCodeAnalysis(absDir)` and print summary to stderr
3. Add import for `"github.com/graphmd/graphmd/internal/code"`
